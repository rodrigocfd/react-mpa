+-----------------------+
| Front-end Siorg React |
+ ----------------------+

-----------------------------------
Subir o ambiente de desenvolvimento
-----------------------------------

Instale os pacotes do Node:
npm i

Suba o servidor http://localhost:3000 de desenvolvimento:
npm start

O Siorg JSF será usado como back-end REST. A aplicação assume que ele está
rodando em http://localhost:8080 .

---------------
Gerar uma build
---------------

A build é composta somente de conteúdo estático: arquivos HTML, CSS e JS
compilados que serão servidos pelo próprio JBoss, da mesma forma que ele serve
outros conteúdos estáticos, como arquivos JPG. Para que o JBoss sirva a build, o
diretório de build deve ser copiado para dentro da área pública do workspace.

O script "gera-e-copia-build.sh" gera a build e já copia para o workspace do
JBoss. No script, altere WORKSPACE_DIR para o caminho onde está seu workspace do
Siorg.

Após isso, o .war gerado irá conter o conteúdo da build.

-----------------------------------
Testar localmente a build compilada
-----------------------------------

Primeiro, rode "gera-e-copia-build.sh".

No Eclipse usando o Navigator, em "siorg-gestao-webapp", clique com o botão
direito e em "Refresh" (ou aperte F5).

Após isto, o diretório build deverá aparecer aqui:

- siorg-gestao-webapp
 - src
  - main
   - webapp
    - public

Em Servers, dê um "Full Publish" no siorg-gestao-webapp, e reinicie o JBoss.

A URL é http://localhost:8080/siorg-gestao-webapp/public/build/index.html

-------------
Chamadas REST
-------------

A aplicação React fará as chamadas REST em:

http://localhost:8080/siorg-gestao-webapp/api/...

Configurável em "producao.config.json".
